# SalesOn 各画面ユーザーストーリー集

---

## 共通フォーマット
- **Actor**: 誰が（営業担当者、マネージャー、サポートスタッフ、AIエージェント）
- **When**: いつ（タイミング）
- **Where**: どこで（画面・場所）
- **What**: 何を（アクション）
- **Why**: なぜ（目的・理由）
- **How**: どのように（方法・手段）
- **UI/明確にすべきこと**: 画面上で明示すべき情報、操作、注意点

---

## 1. AIエージェント画面

### 必須UI要素
- 優先タスク・リスク案件・未完了アクションのリスト/カード
- AI提案・アラート・通知のエリア（色やアイコンで強調）
- AI自動実行履歴・提案履歴の時系列リスト
- 承認・修正・取消などの操作ボタン
- ステータス表示（AI自動/人間承認など）
- サポートスタッフ専用フィルター/ビュー
- 顧客対応履歴とAI提案の紐付け表示
- **入力項目**:
    - Selaへの指示・コマンド入力欄（チャット形式ではなく、エージェントへの指示形式）
    - 承認・却下時のコメント/理由入力欄
    - 顧客対応履歴の追記欄

### ユーザーストーリー1: 状況確認・次アクション提案
- **Actor**: 営業担当者
- **When**: ダッシュボードを開いた時
- **Where**: AIエージェント画面
- **What**: 今日の状況や優先タスク、リスク案件を確認する
- **Why**: 重要な対応漏れやリスクを未然に防ぐため
- **How**: AIが自動で状況を解析し、優先度順に提案・アラート表示
- **UI/明確にすべきこと**:
    - 優先タスク・リスク案件・未完了アクションをカードやリストで明示
    - AI提案と人間の判断・承認の区別を明確に
    - アラート・通知は色やアイコンで視覚的に強調

### ユーザーストーリー2: AI自動実行・履歴確認
- **Actor**: 営業担当者/マネージャー
- **When**: AIが自動でアクションを実行した時、または履歴を確認したい時
- **Where**: AIエージェント画面のアクションパネル
- **What**: AIによる自動実行履歴や提案内容を確認・修正する
- **Why**: AIの判断の透明性確保と誤動作時の修正のため
- **How**: 実行履歴リスト・詳細画面・修正/取消ボタン
- **UI/明確にすべきこと**:
    - AI実行履歴は時系列で表示
    - 各アクションの「AI自動」「人間承認」などのステータス明示
    - 修正・取消操作のガイドや警告表示

### ユーザーストーリー3: サポートスタッフによるAI提案・履歴確認
- **Actor**: サポートスタッフ
- **When**: 顧客からの問い合わせやサポート依頼が発生した時
- **Where**: AIエージェント画面
- **What**: AIによる提案や自動実行履歴を確認し、必要に応じて顧客対応や修正を行う
- **Why**: 顧客対応の品質向上とAIの誤動作防止のため
- **How**: AI提案・履歴リストの確認、修正・承認・却下操作
- **UI/明確にすべきこと**:
    - サポートスタッフ専用のフィルターやビュー
    - 顧客対応履歴とAI提案の紐付け表示
    - 修正・承認・却下ボタンと理由入力欄

## 2. 案件管理画面

### 必須UI要素
- 案件一覧リスト（案件名、顧客名、進捗、リスク、期限、関連タスク・メール）
- 案件詳細パネル
- ステータス・優先度の色分け/バッジ
- 発生源（メール/議事録/AI検知など）の明示
- フィルター・検索機能
- 進捗率・リスク状態のグラフ/アイコン
- コメント・履歴の時系列表示
- サポート案件の識別（タグや色分け）
- サポートスタッフの担当者名表示
- **入力項目**:
    - 案件名・顧客名・説明・期限・優先度の入力/編集欄
    - 進捗率・リスク状態の更新欄
    - コメント・対応履歴の入力欄
    - サポート対応内容の記録欄

### ユーザーストーリー1: 案件一覧・詳細表示
- **Actor**: 営業担当者
- **When**: 案件管理画面を開いた時
- **Where**: 案件管理画面
- **What**: 担当案件の一覧・詳細を確認する
- **Why**: 案件の進捗や対応状況を把握し、次のアクションを決めるため
- **How**: 案件リスト・詳細パネル・フィルター/検索機能
- **UI/明確にすべきこと**:
    - 案件名、顧客名、進捗、リスク、期限、関連タスク・メールを明示
    - ステータスや優先度は色やバッジで視覚化
    - 案件の発生源（メール/議事録/AI検知など）を明示

### ユーザーストーリー2: 案件自動作成・分類
- **Actor**: AIエージェント
- **When**: 新規メール受信や議事録作成時
- **Where**: バックグラウンド/案件管理画面
- **What**: 新規案件を自動で作成・分類する
- **Why**: 対応漏れ防止と業務効率化のため
- **How**: メール・議事録解析→案件自動起票→既存案件との照合
- **UI/明確にすべきこと**:
    - 自動作成案件には「AI自動」ラベル
    - 発生源（メール/議事録/AI検知）を案件詳細に明示
    - 手動作成案件との区別

### ユーザーストーリー3: 案件進捗・リスク管理
- **Actor**: 営業担当者/マネージャー
- **When**: 案件の進捗やリスクを更新・確認する時
- **Where**: 案件管理画面
- **What**: 案件の進捗・リスク状況を管理する
- **Why**: 適切なタイミングでアクションを実行し、リスクを最小化するため
- **How**: 進捗更新・リスクアラート・コメント機能
- **UI/明確にすべきこと**:
    - 進捗率・リスク状態をグラフやアイコンで明示
    - 重要なリスクはアラート表示
    - コメント・履歴の時系列表示

### ユーザーストーリー4: サポートスタッフによる案件進捗・対応履歴管理
- **Actor**: サポートスタッフ
- **When**: 顧客からの問い合わせやサポート案件が発生した時
- **Where**: 案件管理画面
- **What**: 担当案件の進捗や対応履歴を確認・更新する
- **Why**: 顧客対応の抜け漏れ防止と迅速なサポートのため
- **How**: 案件詳細・履歴の確認、進捗・対応内容の記録
- **UI/明確にすべきこと**:
    - サポート案件の識別（タグや色分け）
    - 顧客対応履歴の時系列表示
    - サポートスタッフの担当者名表示

## 3. タスク管理画面

### 必須UI要素
- タスク一覧リスト（タスク名、関連案件、優先度、期限、残り日数、ステータス、AI/手動、承認待ち）
- AI自動生成タスクと手動タスクの区別表示
- 優先度設定・ステータス表示
- ドラッグ&ドロップや優先度ボタン
- 承認待ちタスクの明示、承認・却下ボタン
- 完了時のコメント・次アクション入力欄
- サポートスタッフ専用タスクフィルター
- 顧客対応タスクの明示
- フィードバック・コメント欄
- **入力項目**:
    - タスク名・説明・期限・優先度の入力/編集欄
    - ステータスの更新欄
    - 完了時のコメント・次アクション入力欄
    - 承認・却下時の理由入力欄
    - サポート対応内容・フィードバック入力欄

### ユーザーストーリー1: タスク一覧・優先度管理
- **Actor**: 営業担当者
- **When**: タスク管理画面を開いた時
- **Where**: タスク管理画面
- **What**: 担当タスクの一覧・優先度・進捗を確認・調整する
- **Why**: 今日やるべきことを明確にし、効率的に業務を進めるため
- **How**: タスクリスト・優先度設定・進捗バー
- **UI/明確にすべきこと**:
    - タスク名、関連案件、優先度、期限、残り日数、ステータスを明示
    - AI自動生成タスクと手動タスクの区別
    - ドラッグ&ドロップや優先度ボタンで順序調整

### ユーザーストーリー2: タスク自動生成・承認
- **Actor**: AIエージェント
- **When**: メール受信・議事録作成・案件更新時
- **Where**: バックグラウンド/タスク管理画面
- **What**: 新たなタスクを自動生成し、担当者に通知・承認依頼
- **Why**: 対応漏れ防止と業務効率化のため
- **How**: AI解析→タスク自動抽出→通知・承認フロー
- **UI/明確にすべきこと**:
    - AI自動生成タスクには「AI自動」ラベル
    - 承認待ちタスクは明確に区別
    - 承認・却下ボタンと理由入力欄

### ユーザーストーリー3: タスク実行・進捗更新
- **Actor**: 営業担当者
- **When**: タスクを実行・完了した時
- **Where**: タスク管理画面
- **What**: タスクの進捗を更新・完了報告する
- **Why**: 正確な進捗管理と次アクション計画のため
- **How**: ステータス変更・完了報告
- **UI/明確にすべきこと**:
    - ステータス（未着手/進行中/完了）を明示
    - 完了時のコメント・次アクション入力欄 

### ユーザーストーリー4: サポートスタッフによるタスク対応・進捗管理
- **Actor**: サポートスタッフ
- **When**: サポート関連タスクが割り当てられた時、または顧客対応が必要な時
- **Where**: タスク管理画面
- **What**: 担当タスクの進捗を更新・完了報告し、必要に応じてAIに実行依頼やフィードバックを行う
- **Why**: サポート業務の効率化と品質向上のため
- **How**: タスク進捗更新・完了報告・AI実行依頼・フィードバック入力
- **UI/明確にすべきこと**:
    - サポートスタッフ専用タスクのフィルター
    - 顧客対応タスクの明示
    - フィードバック・コメント欄
